❯ Implement a snake game clone in python3 with pygame. Use venv to create a 
   virtual environment. Rules of the game                                   
  - board is 8x8                                                            
  - snake starts size 3 in the middle, moving right                         
  - there's always an apple on the board; eating it grows the snake by 1    
  - occasionally a star spawns; eating it grows the snake by 3; if uneaten  
  the star disappears after 20 turns; there's only at most 1 star present   
  at every point                                                            
  - the speed of the game increases after every 10 points                   
  - the user can see the current points and speed                           
  - the snake moves with wasd or arrow keys                                 
  - the game starts pressing enter; on game end enter starts a new game     
  - there's no leaderboard or persistence                                   
  - key presses stack - e.g. if the user presses "wd" rapidly (faster than  
  game speed) the snake will turn up on first turn, right on second         
  - self collision is not allowed - turning left when moving right is       
  impossible; this applies to stacked moves as well                         
  - the game starts through a `snake` executable

● Done. Run ./snake to play. Here's the architecture:

  Fractal pattern — state → input → update → render at every scale:
  - Game loop: poll events → accumulate ticks → draw
  - Tick: consume queued direction → move head → check collisions → eat/spawn →
  trim tail
  - Input: raw keypress → validate against last-queued direction → enqueue or
  reject

  Decomposition — 4 independent concerns:
  ┌───────────────┬────────────┬───────────────────────────────────────────────┐
  │    Concern    │   Where    │                   Interface                   │
  ├───────────────┼────────────┼───────────────────────────────────────────────┤
  │ State model   │ State      │ snake, direction, score, growth, phase        │
  │               │ class      │                                               │
  ├───────────────┼────────────┼───────────────────────────────────────────────┤
  │ Input         │ enqueue()  │ validates against queue tail, rejects         │
  │ handling      │            │ opposites                                     │
  ├───────────────┼────────────┼───────────────────────────────────────────────┤
  │ Game logic    │ tick()     │ pure state transition per game tick           │
  ├───────────────┼────────────┼───────────────────────────────────────────────┤
  │ Rendering     │ draw()     │ reads state, writes pixels                    │
  └───────────────┴────────────┴───────────────────────────────────────────────┘
  Key design decisions:
  - Growth counter instead of immediate resize — handles apple (+1) and star
  (+3) uniformly
  - Direction queue with validation against last queued direction — stacked
  moves work correctly, self-collision is impossible
  - Fixed-rate tick accumulator decoupled from render FPS — speed changes don't
  affect input responsiveness
  - Self-collision check accounts for tail vacancy when not growing
