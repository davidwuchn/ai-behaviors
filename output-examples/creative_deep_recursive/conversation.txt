  - board is 8x8

  - occasionally a star spawns; eating it grows the snake by 3; if uneaten  
  the star disappears after 20 turns; there's only at most 1 star present   
  at every point                                                            
  - the speed of the game increases after every 10 points                   
  - the user can see the current points and speed                           
  - the snake moves with wasd or arrow keys                                 
  - the game starts pressing enter; on game end enter starts a new game

  game speed) the snake will turn up on first turn, right on second

  Here's what I built and the key design decisions:

  Architecture: Single-file game (snake_game.py) with a Game class managing
  state, a bash launcher (snake) that auto-creates the venv if missing.

  Tricky bits handled:

  - Input queue with chained validation: Each queued direction is checked
  against the last queued direction (not current), so rapid w then d correctly
  produces up-then-right, while w then s (reversal) is rejected even in the
  queue.
  - Self-collision with tail awareness: When the snake isn't growing, the tail
  will vacate its cell this tick, so the head is allowed to move there. Without
  this, valid moves near the tail get falsely rejected.
  - Growth accumulation: grow counter stacks correctly — eating a star while
  still growing from an apple adds 3 more ticks of retained tail.
  - Star blinking: Stars blink (400ms cycle) when they have fewer than 5 turns
  remaining, giving a visual cue without cluttering the UI.
  - Speed ramping: Moves per second = min(4 + score//10, 15). The game loop
  accumulates fractional time and processes multiple ticks per frame if needed,
  so speed increases never cause dropped moves.
  - Win condition: If the snake fills all 64 cells and no apple can be placed,
  you get a win screen (astronomically unlikely on 8x8 but handles the edge
  case).
